#### 1、redis支持的数据类型

1. String  常见的key-velue
2. List  可以用作消息队列，push和pop方法作为消息推送和消费
3. Set 
4. Zset 
5. Hash 存储对象

#### 2、redis持久化机制

1. RDB持久化方式，定时将某个时间点的数据快照存储在文件中
   1. 只有一个持久化之后的文件，方便恢复
   2. 启动速度快
   3. 容灾性好，可以保存在多个安全的硬盘
   4. 缺点：如果持久化期间出现问题会导致数据丢失
2. AOF持久化方式，记录所有redis命令到aof文件
   1. 可以记录所有的操作命令
   2. 如果中途服务器宕机，也不怕数据丢失
   3. 缺点：文件大，恢复速度慢
   4. 缺点：数据集大的时候启动慢

#### 3、redis过期键的清除方式

1. 定时删除：创建的时候同步创建计时器
2. 惰性删除：等到被调用的时候，检查是否过期，如果过期则删除
3. 定期删除：定期遍历数据库，对过期的键删除
